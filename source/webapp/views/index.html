Introduction

This website is built by Nghiem Tien Dat, a student from faculty of Information Technology, Ton Duc Thang University. It is a demo in Information Technology Project course, with the guidance of Mr. Doan Xuan Thanh.

The goal of website is to explore high occupancy itemsets by using two algorithm:

  • Deep first search for mining high occupancy itemsets (DFHOI).
  • Auto-thresholding for mining high occupancy itemsets (ATHOI).

DFHOI was introduced by Loan T.T. Nguyen and other co-authors in 2022. It uses upper-bound occupancy and deep first search to discover high occupancy itemsets. However, you may have difficulty choosing the most suitable minimum occupancy threshold to get the right number of itemsets.

Therefore, we introduce a new algorithm based on DFHOI. Unlike the previous one, you just need to determine how many high occupancy itemsets you would like to explore. It will be more convenient and widely applied in practice. However, with massive datasets, ATHOI algorithm takes more time and resources than DFHOI.

What functions does website provide?

The website has two main features:

  • Mine high occupancy itemsets with minimum occupancy threshold.
  • And, find top k itemsets that have highest occupancy values.

The former is implemented according to ATHOI algorithm, and the latter is implemented according to DFHOI algorithm.

You can easily use our functionalities by clicking tags on the navigation bar. Then, you need to fill in required information such as source file, minimum occupancy threshold, or the number of itemsets to mine.

If you have difficulty doing so, please read the instructions on the right.

Format of source file

The source file is a simple text file recording transaction database. In which, each line represents a transaction and each item in a particular transaction is separated by a space character.

Format of returned file

The returned file is also a text file containing high occupancy itemsets found. In which, each line has an itemset and its corresponding occupancy value. Itemsets are enclosed in parentheses and each item is separated by a comma.

Tips for you

  • The maximum file size allowed to upload is 2MB.
  • If the processing time exceeds 5 minutes, it will be terminated.
  • The returned file is stored in server during 5 minutes. After that, it will be deleted and you can not download it anymore.
  • Be careful with refreshing page because the returned file will disappear.
  • Choose an appropriate the number of itemsets or minimum occupancy threshold for saving time and resources.

How to find top k high occupancy itemsets?

  1. Select source file to upload (maximum 2MB).
  2. Choose the number of itemsets (between 1 and 100). It will return all in case the given number is greater than the number of itemsets in file.
  3. Click on "Mine" button to get started.
  4. Wait for system to process data. It will be terminated if the duration is over 5 minutes.
  5. After processing successfully, click on file to download.
  6. If there is an error, please read the instructions and try again.

How to mine high occupancy itemsets?

  1. Select source file to upload (maximum 2MB).
  2. Choose the suitable minimum occupancy threshold (between 0 and 1). It will return all itemsets in case this threshold is 0.
  3. Click on "Mine" button to get started.
  4. Wait for system to process data. It will be terminated if the duration is over 5 minutes.
  5. After processing successfully, click on file to download.
  6. If there is an error, please read the instructions and try again.
